A Counterintuitive Product Decision

当市面上每一款 AI 编程工具都在争夺 IDE 的地盘——Cursor 做了一整个编辑器，Copilot 嵌进了 VSCode 的骨髓，Windsurf 也在走同样的路——Anthropic 的 Claude Code 选择了一个看似荒谬的方向：纯终端。

没有语法高亮，没有文件树，没有侧边栏。一个黑底白字的命令行界面。

这不是倒退，而是一个精确的设计取舍。终端是 Unix 哲学的原生载体：可组合、可脚本化、可嵌入 CI/CD pipeline。当你的 AI 助手栖息在终端中，它天然就获得了对操作系统的完整访问权限——文件系统、shell 命令、git 操作、环境变量。它不再是一个在侧边栏里提供建议的顾问，而是一个能亲手执行任务的代理（Agent）。

这个区别是本质性的。

While every AI coding tool competes for IDE territory — Cursor built an entire editor, Copilot embedded itself in VSCode's core, Windsurf follows the same path — Anthropic's Claude Code chose a seemingly absurd direction: the bare terminal.

No syntax highlighting, no file tree, no sidebar. A black-and-white command-line interface.

This is not regression but a precise design tradeoff. The terminal is the native carrier of Unix philosophy: composable, scriptable, embeddable in CI/CD pipelines. When your AI assistant inhabits the terminal, it gains full operating system access by nature — filesystem, shell commands, git operations, environment variables. It ceases to be an advisor offering suggestions in a sidebar and becomes an agent capable of executing tasks firsthand.

This distinction is fundamental.

Core Commands

以下命令构成日常操作的基本词汇表：

命令功能
/help列出所有可用命令
/cost查看当前会话的 token 消耗与费用
/compact压缩对话历史，释放上下文空间
/model切换模型（Opus / Sonnet / Haiku）
/clear清空当前对话，重置上下文
Esc取消正在编辑的输入
Ctrl+C中断当前操作或退出
其中 /cost 和 /compact 是使用频率最高的两个。前者提供实时的费用感知，后者在对话变长时压缩上下文以遏制 token 膨胀。两者的具体用法将在本系列"省钱篇"中展开。

The following commands form the essential vocabulary for daily operations:

CommandFunction
/helpList all available commands
/costView token consumption and cost for the current session
/compactCompress conversation history to free context space
/modelSwitch models (Opus / Sonnet / Haiku)
/clearClear the current conversation and reset context
EscCancel the input being edited
Ctrl+CInterrupt current operation or exit
/cost and /compact see the highest usage frequency. The former provides real-time cost awareness; the latter compresses context as conversations grow to curb token inflation. Detailed usage for both is covered in the "cost optimization" post in this series.

Permission Model

Claude Code 提供三级权限控制，本质上是"自主性"和"安全性"之间的滑动标尺：

1. 交互确认模式（默认）

每次文件编辑、命令执行等操作都需要用户逐一审批。安全性最高，但频繁的确认提示会打断工作节奏。适合初次使用或处理敏感项目。

2. 自动编辑模式

文件读写操作自动执行，shell 命令仍需确认。这是效率与安全之间的帕累托最优点。启动时指定允许的工具集：

Claude Code offers three tiers of permission control — essentially a sliding scale between autonomy and safety:

1. Interactive Confirmation (Default)

Every file edit, command execution, and similar operation requires individual user approval. Maximum safety, but frequent confirmation prompts interrupt workflow. Best for initial use or sensitive projects.

2. Auto-Edit Mode

File read/write operations execute automatically; shell commands still require confirmation. This is the Pareto-optimal point between efficiency and safety. Specify allowed tools at launch:

3. 全自动模式

所有操作无需确认直接执行。仅建议在以下条件同时满足时使用：任务边界清晰、项目处于 git 版本控制之下、你对当前操作有充分预期。

3. Full Autonomy Mode

All operations execute without confirmation. Only recommended when the following conditions are simultaneously met: the task boundary is clear, the project is under git version control, and you have full expectation of the operations involved.

命令名本身就是一句警告。日常开发中，自动编辑模式足以覆盖绝大多数场景。

The command name itself serves as a warning. For daily development, auto-edit mode covers the vast majority of scenarios.

配置就绪之后，Claude Code 的价值体现在四类典型任务上。

Paradigm 1: Codebase Comprehension

Claude 会自主搜索文件、追踪依赖、分析调用关系，最终给出结构化的架构概览。这个过程对人来说可能需要数小时，对 Claude 来说通常在一分钟以内。

范式二：定向修复

Claude autonomously searches files, traces dependencies, and analyzes call relationships, ultimately delivering a structured architectural overview. What might take a human hours typically takes Claude under a minute.

Paradigm 2: Targeted Fixes

Feed error messages directly as input:

Claude 不只是诊断问题——它会定位代码、分析根因、直接修改文件。从"建议你怎么改"到"已经帮你改好了"，这是 AI 编程工具从顾问到代理的质变。

范式三：功能实现

Claude does not merely diagnose — it locates the code, analyzes root causes, and modifies the files directly. The leap from "suggesting what to change" to "having already changed it" marks the qualitative shift of AI coding tools from advisor to agent.

Paradigm 3: Feature Implementation

这是 Claude Code 投入产出比最高的场景。它会读取现有代码以理解项目规范，编写符合规范的新代码，并同步更新相关的测试文件。你的角色从"编码者"转变为"需求定义者与审查者"。

范式四：批量重构

This is Claude Code's highest-ROI scenario. It reads existing code to understand project conventions, writes convention-compliant new code, and synchronously updates related test files. Your role shifts from "coder" to "requirements definer and reviewer."

Paradigm 4: Batch Refactoring

批量重构对人类而言是高认知负荷、低创造性的苦差事——恰好是 AI 代理最适合接管的任务类型。Claude Code 能在保持代码风格一致的前提下完成全量修改，且不会遗漏。

Batch refactoring imposes high cognitive load with low creativity on humans — precisely the task type best suited for AI agent takeover. Claude Code completes full-scope modifications while maintaining code style consistency, with no omissions.

CLAUDE.md: The Prerequisite for All Efficiency

在项目根目录放置一个 CLAUDE.md 文件。Claude Code 每次启动时自动读取它，将其作为项目上下文注入对话。这份文件的质量直接决定了 Claude 的输出质量——它是否了解你的技术栈、代码规范、目录结构、禁忌操作。

Place a CLAUDE.md file in your project root. Claude Code reads it automatically on every launch, injecting it as project context into the conversation. The quality of this file directly determines Claude's output quality — whether it understands your tech stack, coding conventions, directory structure, and forbidden operations.

An Honest Statement of Limitations

Claude Code 不适合以下场景：需要深层领域知识的架构决策、对业务逻辑正确性有严格要求的核心路径、需要实时 UI 反馈的前端开发。它的优势区间非常明确——代码理解、生成、修改和命令执行——而在这个区间内，它是当前最高效的工具形态。

终端界面起初可能令人不适。但当你适应了"用自然语言操控操作系统"这一交互范式之后，回到手动操作会显得格外笨拙。

Claude Code is unsuited for: architecture decisions requiring deep domain knowledge, critical paths with strict business logic correctness requirements, and frontend development needing real-time UI feedback. Its advantage zone is clearly defined — code comprehension, generation, modification, and command execution — and within that zone, it represents the most efficient tool form available.

The terminal interface may feel uncomfortable initially. But once you adapt to the interaction paradigm of "directing an operating system with natural language," returning to manual operations feels remarkably clumsy.
