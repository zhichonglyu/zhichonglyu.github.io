An Overlooked Critical Variable

大量用户在安装 Claude Code 两三天后得出"不过如此"的结论。典型症状：生成的代码风格与项目格格不入，测试命令选错了包管理器，新文件放进了错误的目录。

这些失败几乎都指向同一个根因：Claude 缺乏项目上下文。

Claude Code 在启动时自动加载项目根目录下的 CLAUDE.md 文件。这份文件的本质是一个隐式知识的显式化容器——把团队成员"默认都知道"但从未写下来的规则，转化为 Claude 可以消费的结构化输入。

写好这个文件，输出质量立即跃升。不写，则 Claude 只能依赖统计推断来猜测你的项目规范——而统计推断在面对项目特异性时，错误率令人沮丧。

A significant number of users conclude Claude Code is "nothing special" after two or three days. Typical symptoms: generated code clashes with project style, test commands invoke the wrong package manager, new files land in incorrect directories.

These failures almost invariably trace to the same root cause: Claude lacks project context.

Claude Code automatically loads a CLAUDE.md file from the project root on startup. This file is essentially a container for making implicit knowledge explicit — converting the rules that team members "just know" but never documented into structured input Claude can consume.

A well-crafted file produces an immediate leap in output quality. Without it, Claude relies on statistical inference to guess project conventions — and statistical inference, when facing project-specific idiosyncrasies, yields frustrating error rates.

Three-Tier Configuration

CLAUDE.md 并非只有项目根目录一个挂载点。Claude Code 的配置系统分为三个层级，遵循"就近优先"原则：

1. 用户级 ~/.claude/CLAUDE.md

全局生效，对所有项目共享。适合存放个人维度的通用偏好——编程语言、沟通风格、提交习惯：

CLAUDE.md is not limited to a single mount point at the project root. Claude Code's configuration system spans three tiers, following a "nearest-first" priority:

1. User-level ~/.claude/CLAUDE.md

Globally effective, shared across all projects. Suitable for personal preferences — programming language, communication style, commit habits:

2. 项目级 项目根目录/CLAUDE.md

最核心的一层。项目技术栈、代码规范、目录约定、安全禁忌——所有项目特异性信息的归属地。

3. 目录级 子目录/CLAUDE.md

面向 monorepo 或模块化架构。前端和后端使用不同的代码规范？分别在 frontend/CLAUDE.md 和 backend/CLAUDE.md 中声明。

三个层级在运行时合并生效。当存在冲突时，更具体的层级覆盖更通用的层级——与 CSS 的级联规则类似。

2. Project-level project-root/CLAUDE.md

The most critical tier. Project tech stack, code conventions, directory structure, security constraints — the home for all project-specific information.

3. Directory-level subdirectory/CLAUDE.md

Designed for monorepos or modular architectures. Frontend and backend use different coding conventions? Declare them separately in frontend/CLAUDE.md and backend/CLAUDE.md.

All three tiers merge at runtime. When conflicts exist, more specific tiers override more general ones — similar to CSS cascade rules.

What to Include: Five Core Modules

1. Tech Stack Overview

无需面面俱到——Claude 能通过读取代码自行补全细节。此处的目标是提供顶层架构决策，帮助 Claude 建立正确的心理模型。

No need to be exhaustive — Claude can fill in details by reading the code. The goal here is to provide top-level architectural decisions that help Claude build the correct mental model.

2. Command Registry

这个模块的价值被严重低估。没有它，Claude 面临一个 npm / pnpm / yarn / bun 的四选一猜测题，误判率极高。明确声明命令可以消除这一整个类别的错误。

This module's value is severely underestimated. Without it, Claude faces a four-way guessing game among npm / pnpm / yarn / bun, with a high error rate. Explicit command declarations eliminate this entire category of mistakes.

3. Code Conventions

这是影响日常体验最深的模块。缺少这些规范声明，Claude 会退化到"统计上最常见的写法"——生成 export default function 而不是 named export，使用 throw new Error() 而不是自定义错误类。单次修正耗时不过几秒，但日积月累是惊人的摩擦成本。

This is the module with the deepest impact on daily experience. Without these convention declarations, Claude degrades to "statistically most common patterns" — generating export default function instead of named exports, using throw new Error() instead of custom error classes. Each correction takes only seconds, but the cumulative friction cost is staggering.

4. Directory Structure

列出关键目录及其职责即可。Claude 据此判断新文件的归属位置。

List key directories and their responsibilities. Claude uses this to determine where new files belong.

5. Constraint Declarations

"不要做什么"的信息密度远高于"要做什么"。自动生成的目录、包含密钥的文件、已废弃的模块——这些"地雷"只有通过显式声明才能让 Claude 规避。仅靠 Claude 自行推断，踩雷只是时间问题。

"What not to do" carries far higher information density than "what to do." Auto-generated directories, files containing secrets, deprecated modules — these "landmines" can only be avoided through explicit declaration. Relying on Claude's inference alone, stepping on one is merely a matter of time.

Anti-Patterns: Four Categories to Avoid

冗余信息。"这是一个 JavaScript 项目"——package.json 的存在已经传达了这一信息。CLAUDE.md 应当只包含不可从代码中推断的知识。 过度篇幅。CLAUDE.md 在每轮对话中都作为系统上下文被消费。一份千行级的文档意味着每条消息开头就要消耗数千 token。控制在 100-200 行以内是一个合理的上限。 易变内容。"当前正在开发用户注册功能"——这类状态信息的半衰期以天计。将其留给对话本身，而不是固化到配置文件中。 可推断的细节。函数签名、API 参数等信息 Claude 可以直接从源代码中获取，无需在 CLAUDE.md 中复述。
Redundant information. "This is a JavaScript project" — the existence of package.json already communicates this. CLAUDE.md should contain only knowledge that cannot be inferred from the code. Excessive length. CLAUDE.md is consumed as system context in every conversation turn. A thousand-line document means thousands of tokens consumed at the start of each message. 100-200 lines is a reasonable upper bound. Volatile content. "Currently developing user registration feature" — such state information has a half-life measured in days. Leave it to the conversation itself rather than crystallizing it in configuration. Inferable details. Function signatures, API parameters — Claude can obtain these directly from source code without CLAUDE.md repeating them.

Quantified Impact

基于个人的持续使用观察：配置了完善的 CLAUDE.md 之后，Claude 首次输出即符合项目规范的比率从约 40-50% 提升至约 80%。提升幅度看似有限，但其经济学意义值得细算——每一次"不对，我们用 pnpm 不用 npm""这个文件应该放在 services 目录下"的纠正，都是一次额外的 API 调用，消耗的 token 远超那行纠正文本本身。

花 20 分钟写一份好的 CLAUDE.md，其收益在此后的每一次对话中持续兑现。从投入产出比的角度看，这大概是 Claude Code 使用流程中回报率最高的单项投资。

Based on sustained personal observation: after configuring a thorough CLAUDE.md, the rate at which Claude's first output conforms to project conventions rose from approximately 40-50% to roughly 80%. The improvement appears modest, but its economic significance merits analysis — each correction like "no, we use pnpm not npm" or "this file belongs in the services directory" constitutes an additional API call, consuming tokens far exceeding the correction text itself.

Spending 20 minutes writing a good CLAUDE.md yields returns in every subsequent conversation. From an ROI perspective, this is likely the highest-return single investment in the Claude Code workflow.
